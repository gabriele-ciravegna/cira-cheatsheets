BuildPartner SaaS plugin

AI build partner per Claude Code. Expert frameworks, coaching, training personalizzato — pricing, content, launch, AI automation, marketing, sales, product. Dashboard + Skill usage tracking.

Quando usarlo: serve framework battle-tested invece di risposta generica. Pricing, content strategy, launch, marketing decisions con expert backbone.
NON usarlo se: Vuoi solo dev tool tecnico (BuildPartner è business/strategy oriented). Free tier limitato, dopo è a pagamento Stripe.

Quick Start

# Marketplace
claude plugin marketplace add austinmarchese/buildpartner-plugin
claude plugin install buildpartner@buildpartner

# Prima volta: crea account, apre dashboard, registra MCP

# Update
claude plugin marketplace update
claude plugin update buildpartner@buildpartner

# Uninstall
claude plugin uninstall buildpartner@buildpartner

Dashboard: buildpartner.ai/dashboard — skill usage, quest progress, AI profile, recap sessioni.

Architettura

Quattro pezzi:

  • Skills (plugin/skills/) — slash command route al framework giusto
  • MCP server (buildpartner su npm) — espone get_expert_knowledge
  • Hook (plugin/hooks/) — session-boundary gating, usage flush, auto-update, reminder weekly
  • Web app (buildpartner.ai) — dashboard, account, payments, expert knowledge API

MCP server avviato da Claude Code via npx -y buildpartner@latest serve. Auto-update check a session start + end.

Gotchas & Anti-pattern

  • Free tier limitato — dopo paga o stop. NON è freeware permanente.
  • Dashboard cookie/session — sign-in automatico via plugin, ma scade. Re-login se /bp:open-dashboard fallisce.
  • Expert framework può sovrappore mentori CIRA — scegli source autorevole (memo) per evitare doppi.
  • Auto-update plugin a session start — può rallentare cold start.
  • MCP server get_expert_knowledge richiede internet — offline = degraded.
