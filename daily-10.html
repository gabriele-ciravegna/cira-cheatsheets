I 10 essenziali

/costruiamo X
tool: CIRA Customuso: build nuovo
Cosa faTrigger build progetto. Intervista 10-15 domande → propone path A1-A5 + execute
QuandoInizio progetto. "facciamo l app", "facciamo il sito", "lets build X"
Esempio/costruiamo agente telegram gestione email
See alsocira-custom · GSD · gstack
/gsd-progress
tool: GSDuso: capire dove sei
Cosa faMostra stato phase corrente, prossimi task, progress milestone
Quando"Non so cosa fare adesso", riapri sessione, post /clear
Esempio/gsd-progress
See also/gsd-resume-work · /gsd-next · /gsd-stats
claude --resume
tool: Claude Codeuso: continuare
Cosa faRiprende ultima sessione Claude Code della cwd
QuandoApri terminale al mattino su progetto attivo, post crash, post chiusura PC
Esempiocd ~/WORK/SOFTWARE/myproject && claude --resume
See alsoclaude (nuova) · /clear · /compact
orchestra (skill)
tool: CIRA Customuso: routing
Cosa faRouter multi-skill: query complessa → mostra rete neurale + invoca skill correlate
QuandoCoaching, pricing, decisione bloccato, query multi-aspetto
Esempio"sono bloccato su pricing cliente X" → orchestra invoca aristotle + ali-pricing + hormozi
See alsocira-custom
tg-reply <chat_id> <msg_id> <text>
tool: Bash + Telegramuso: rispondere veloce
Cosa faRisposta Telegram via helper bash al bot launchd (com.cira.telegram-bot)
QuandoReplicare a messaggio da terminale senza aprire MCP
Esempiotg-reply 1689116477 220 "fatto, deploy ok"
See alsolaunchctl list | grep telegram
launchctl list | grep cira
tool: Bash macOSuso: healthcheck
Cosa faLista servizi launchd CIRA con PID + last exit code
QuandoPost-blackout, "il bot Telegram non risponde", debug Aurelio coach
Esempiolaunchctl list | grep cira ; launchctl print gui/$(id -u)/com.cira.coach-bot | grep state
See alsobash-macos processes section
MAI pkill servizi launchd — usa launchctl bootout/bootstrap
curl -sS -m 5 -o /dev/null -w 'HTTP:%{http_code}\n' URL
tool: Bash macOSuso: healthcheck endpoint
Cosa faVerifica endpoint vivo, ritorna solo HTTP status code
QuandoTest :37777 claude-mem worker, Vercel deploy, API esterne
Esempiocurl -sS -m 5 -o /dev/null -w 'HTTP:%{http_code}\n' http://localhost:37777
See alsolsof -iTCP:PORT (chi listen sulla porta)
git push origin main
tool: Gituso: deploy
Cosa faPush main → trigger automatico Vercel deploy (cira-dashboard, mercurio)
QuandoDopo commit. Sempre con git status + diff prima
Esempiogit status && git diff && git push origin main
See alsogh pr create · gh run watch
MAI force push su main. Sempre verifica deploy Vercel post-push (rare 5xx)
