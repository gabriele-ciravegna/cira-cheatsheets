Claude Code v2.x · 2026-05-15

Anthropic CLI ufficiale per development in Claude Code

Quando usarlo: Development locale con Claude AI editor, debugging, testing, multi-file refactoring, multi-agent orchestration.

Quick Start

Install

npm install -g @anthropic-ai/claude
# o via brew (se disponibile)
brew install anthropic/claude/claude

Verify & Update

claude --version
npm update -g @anthropic-ai/claude

Richiede: Node.js 18+, npm 8+

First run

cd ~/WORK/SOFTWARE/myproject
claude
# REPL interattivo. Carica CLAUDE.md della dir corrente + .claude/settings.json

Headless one-shot

claude -p "fix bug in src/auth.ts" --output-format json

Riprendi ultima sessione

claude --resume

Hook Types

HookQuando scattaUso tipico
SessionStart All'inizio di ogni sessione Claude Code Fetch dashboard, arm monitors, init cron, inject memory
PreToolUse Prima dell'esecuzione di un tool Validare params, loggare call, bloccare tool non autorizzati
PostToolUse Dopo l'esecuzione di un tool Auto-format output, run linter, trigger alert, verify result
Stop Quando la sessione termina Commit changes, push backup, save state, final checks
UserPromptSubmit Prima che un prompt utente venga processato PII filter, routing priorità, downgrade a Haiku
Custom hook User-defined (project-specific) Deploy trigger, test gate, automazioni dominio specifico

Advanced Patterns — Dettaglio

Extended thinking: Default ON — usa Option+T per toggle. Budget fino a 31,999 token di ragionamento interno. Abilita per task complessi; disabilita per edit singoli file.
Plan Mode: Ctrl+G attiva un workflow strutturato multi-step. Claude mostra il piano completo prima di eseguire. Ideale per implementazioni multi-fase.
Git worktree isolation: EnterWorktree tool crea branch isolato. Permette refactoring complesso senza rischiare il main. Usa ExitWorktree per tornare.
Self-Evolving pattern: dopo 10+ sessioni sullo stesso progetto, usa /evolve. Analizza corrections.jsonl + observations.jsonl e promuove pattern ricorrenti a regole permanenti in learned-rules.md.
CLAUDE.md locale: file in .claude/CLAUDE.md nella root del progetto agisce come system prompt locale. Override delle impostazioni globali. Metti qui: stack tech, regole naming, agenti predefiniti.
Skill evolution: /evolve può creare nuove skill da pattern di correzione. Output in ~/.claude/skills/. Ciclo: errore → correzione → jsonl → /evolve → skill permanente.
Parallel agents — sweet spot: 3-5 agenti in parallelo ottimale. Oltre 8 si degrada la qualità. Sub-agents via Task() isolano il contesto ma il summary è lossy. Hybrid pattern: main thin + sub-agent specialist.

Common Workflows

Feature implementation

  1. Research GitHub/docs
  2. Plan con /plan agent
  3. TDD con /tdd
  4. Code review /code-review
  5. Commit + push

Bug fix rapid iteration

  1. /clear context
  2. Isolate con test
  3. /tdd RED → GREEN → IMPROVE
  4. Verify coverage ≥ 80%
  5. Commit

Refactoring large codebase

  1. EnterWorktree for isolation
  2. Map codebase /gsd:map-codebase
  3. Plan phases /plan
  4. Execute by phase /gsd:execute-phase N
  5. Review /code-review
  6. Merge

Multi-agent decision (C-suite)

  1. TeamCreate con 4-5 agents
  2. Assegna ruoli: security, perf, cost, UX, architect
  3. Moderator prompts decision
  4. Agents debate + vote
  5. Record decision in CLAUDE.md

Long research + parallel tasks

  1. Task 1 con run_in_background
  2. Task 2 parallel
  3. Task 3 parallel
  4. Monitor via TaskCreate tracking
  5. Synthesis quando tutti completati

Project initialization

  1. claude project new <name>
  2. Install rules + hooks (claude-init)
  3. Set CLAUDE.md defaults
  4. Create .claude/skills/ structure
  5. Push git init commit
