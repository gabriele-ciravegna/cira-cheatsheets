Ruflo 3.7.0-alpha.23

Sandbox parallel iterations + scoring + promote winner. Per scelte design-critical con coordinamento swarm automatico

Quando usarlo:
  • UX/UI ambigua con N approcci alternativi
  • Scelte design-critical che richiedono A/B di varianti parallele
  • Iterazione rapida con scoring automatico e promozione winner
  • Orchestrazione multi-agent su design problem complesso
NON usarlo se:
  • Soluzione obvia o deterministica
  • Deadline urgente (setup init richiede ~2 minuti)
  • Repository monolitico senza isolamento sandbox
  • Ambito limitato a single-agent execution

Quick Start

Install
npm install -g ruflo
Binary: /Users/cira/.local/node-v22.14.0-darwin-arm64/lib/node_modules/ruflo
Verifica: ruflo --version
Pipeline base
ruflo init --full --start-all → inizializza tutto + avvia daemon/memory/swarm
ruflo task create -t design -d 'N varianti' → definisci task paralleli
ruflo swarm init --v3-mode && ruflo swarm start -o 'obiettivo' -s development → avvia competition
ruflo analyze code --format json → score ogni variante
ruflo task list --all → review + pick winner
Performance metrics
Init: ~2 min  ·  Parallel agents: up to 15 (V3 mesh)  ·  Memory search: 150x–12,500x faster (HNSW)  ·  Consensus: <500ms/round  ·  Task create: <100ms  ·  Analyze 10k LOC: <5s

Workflows End-to-End

Sandbox Design Competition — 5 varianti UI card layout
Tempo stimato: 5–15 minuti per 5 varianti parallele

1. ruflo init --full --start-all
2. ruflo task create -t design -d 'Generate 5 card layouts: minimal/bold/glass/shadow/flat'
3. ruflo swarm init --v3-mode
4. ruflo swarm start -o 'Design card variations' -s development --agents 5
5. (agents iterate in parallel, each generates variant)
6. ruflo analyze code --format json → score each variant: complexity, readability, patterns
7. ruflo memory store → best patterns from winner variant
8. ruflo task list --all → review all variant submissions
9. Manual promotion: pick winner → implement in main codebase
Iterazione Rapida con Memory Learning

1. ruflo session resume session-ui-variants-001
2. ruflo memory search 'card layout patterns from prior sessions'
3. Nuovo task con insights precedenti pre-loaded
4. ruflo swarm coordinate --agents 15 → use learned patterns
5. ruflo analyze diff --risk → valuta incrementale changes
6. Promote vincitore + persist new patterns
Merge con /costruiamo workflow

1. /costruiamo trigger con path 'ruflo-only'
2. Ruflo init inline → crea sandbox temporaneo
3. Task definition from /costruiamo phase1
4. Parallel agent iterations + scoring
5. Results fed back a /costruiamo phase2 (tdd-guide, code-reviewer)

Gotchas & Tips

Ruflo daemon processes run via launchd on macOS — mai pkill
FixUsa ruflo daemon stop o launchctl stop command
Init crea .claude-flow/ directory in repo root — non gitignore se vuoi condividere config
FixReview .gitignore, decide se .claude-flow/ è shareable (env vars mantenere separate)
Memory vector DB starts fresh dopo init; occorrono ~10 task per buon learning signal
FixImporta patterns da previous sessions con ruflo session resume o ruflo memory recall
15-agent mesh coordination ha overhead latency; per quick decision usare --agents 3-5
FixBalance parallelism vs consensus latency dipende dall’obiettivo
TipSempre esegui ruflo analyze su output variation prima di manual scoring — objective metrics + early detection red flags (complexity, circular deps)
TipRuflo memory search è semantic, non literal — cerca pattern type, non stringa esatta
Esempioruflo memory search 'layout patterns for cards under 200px width'
