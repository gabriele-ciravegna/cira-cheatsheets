Compound Engineering v3.8.2

Review-driven development plugin. Ogni ciclo migliora i successivi: 26 reviewer agent in parallelo → learnings → COMPOUND.md registry → prossimo ciclo più veloce.

Namespace: /compound-engineering:ce-*  ·  Rilasciato: 2026-05-14

Quick Start

Plugin Claude Code v3.8.2. Trigger entrypoint quotidiano:

/compound-engineering:ce-work

Comandi via prefisso /compound-engineering:ce-*. 48 specialist agent disponibili per review e research.

Core principle: Multi-persona review (correctness, security, testing, maintainability, performance, domain expertise) → learnings → COMPOUND.md registry → next cycle faster.
Quando usare CE: Ogni feature, PR, bug fix, ottimizzazione. Integra con /gsd, gstack, /costruiamo.

Review

/compound-engineering:ce-code-review
review Mandatory pre-PR 26 agent in parallelo
Cosa fa Multi-persona code review. Spawns 26 specialized reviewer agents (syntax, security, testing, perf, maintainability, domain). Returns structured verdict + learnings.
Input
file paths code snippet context (optional)
Output
agent verdicts (pass/warn/block) learnings.md entries commit-msg suggestions
Headless mode v3.8.0+. Base detection v3.8.1+ (prose-driven). Usa ce-compound dopo la review.
/compound-engineering:ce-pr-audit
review Pre-push gate
Cosa fa Full PR audit: code review + changelog scan + dependency check + test coverage.
Input
PR branch target branch
Output
compliance matrix blocker list merge readiness score
Agent-native compliance. Outputs HTML report + jq-valid JSON.
/compound-engineering:ce-document-review

Cosa fa 7-persona review di docs: clarity, completeness, grammar, structure, SEO, localization, accessibility.
Input

Scans .claude/rules/ per terminology standards.

Workflows end-to-end

Feature Delivery Cycle — ciclo principale CE
  • ce-plan
    phase plan + COMPOUND.md lookup
  • ce-worktree-init
    feature branch + audit dir
  • code impl + commit loop
    git commit → ce-git-workflow check
  • ce-code-review
    26-agent verdicts + learnings draft
  • ce-compound
    COMPOUND.md patch + next-cycle improvements
  • ce-pr-audit
    compliance matrix + merge readiness
  • push + merge
PR Review Gates — multi-cycle improvement: early PRs slower, later PRs faster (COMPOUND.md learnings accumulate)
ce-code-review
Blocker: verdicts da ce-reviewer-security, ce-reviewer-testing
ce-pr-audit
Blocker: coverage <80% OR compliance score <50
ce-git-workflow
Blocker: commit msg non-conforming OR audit dir pollution
Bug Fix Workflow
  • ce-compound query
    find prior similar bugs + fix patterns
  • ce-code-review
    review on fix candidate
  • ce-compound
    harvest learnings
Optimization Loop
  • ce-performance-profile
    identify bottleneck
  • ce-code-review
    on optimization candidate
  • ce-compound
    extract pattern for next optimization
Legacy Rescue
  • ce-agent-native-audit
    assess agentic readiness
  • ce-refactor-guide
    phase-based cleanup roadmap
  • per phase: ce-code-review + ce-compound
Team Onboarding
  • ce-learning-export
    export COMPOUND.md learnings
  • ce-workflow-detect
    show optimal CE workflow per il team
  • ce-onboard-agent (optional)
    teach CE new custom agent

Stack Integration

/gsd

ce-plan inputs from GSD phase plan. ce-code-review runs after feature impl phase. ce-compound outputs patch per GSD learnings doc.

gstack

ce-code-review integra con gstack test suite (ce-test-coverage-audit). Parallel review agents + gstack parallel test runners.

/costruiamo

ce-audit-cira checks /costruiamo output. ce-onboard-agent insegna CE nuovi custom agent. /costruiamo triggers ce-compound come learnings harvest step.

superpowers

ce-compound feeds superpowers research synthesis (reuse patterns, proven approaches).

Gotchas

26 agent in parallelo: ce-code-review spawna 26 agent simultanei. In CI usa --headless + setta un concurrency limit per evitare rate limit.
COMPOUND.md = advisory, non enforced: ce-compound surfaces patterns, il developer decide l'adozione. I learnings non si auto-applicano.
ce-plan reads, non auto-apply: Legge COMPOUND.md ma NON applica automaticamente. Richiede decisione manuale + doc closure.
ce-audit-cira richiede struttura prevedibile: L'output di /costruiamo deve avere skill name + file path standard. Output non-standard → audit fails.

FAQ rapide

Posso runnare ce-code-review senza tutti i 26 agent?
Si. --agents=security,testing,perf restringe lo scope. Ma la review a 26 agent completa cattura molto di piu'.
ce-compound auto-mergia COMPOUND.md?
No. Outputs un patch candidate. Review + merge manuale (o via CI gate).
ce-code-review funziona in CI senza user prompts?
Si. Flag --headless + JSON output. Integra in GitHub Actions / pre-commit hook.
COMPOUND.md ha size limits?
Nessun limite stretto, ma splitta per category/year se >10MB. ce-learning-export aiuta a gestire.
ce-code-review e COMPOUND.md si contraddicono?
I verdetti real-time vincono. COMPOUND.md e' storico. Real-time wins; aggiorna COMPOUND.md se i learnings sono cambiati.
