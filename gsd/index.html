Quick

Daily 10 ↗ Workflows CIRA ↗

GSD v1.42.2

Spec-driven development + context engineering. Anti-context-rot. Fase → discussion → planning → execution → verification → shipping.

Quando usarlo: Progetto non-banale (≥3 file), multi-fase, audit trail richiesto, team collaboration
NON usarlo se: Bug isolato (→ /gsd:debug standalone), task triviale (→ /gsd:quick), refactor file singolo (→ /refactor-clean)

Quick Start

Install

npx -y --package=get-shit-done-cc@latest -- get-shit-done-cc --claude --global

Update esistente

/gsd:update

Uninstall

rm -rf ~/.claude/get-shit-done ~/.claude/commands/gsd* ~/.claude/agents/gsd-* ~/.claude/skills/gsd-*

Default workflow

/gsd:new-project → /gsd:discuss-phase 1 → /gsd:plan-phase 1 → /gsd:execute-phase 1 → /gsd:verify-work 1 → /gsd:ship

Workflow End-to-End

Greenfield project completo
/gsd:new-project → /gsd:discuss-phase → /gsd:plan-phase → /gsd:execute-phase → /gsd:verify-work → /gsd:ship
Tempo stimato: 3-8h per phase, 1-2h per workflow — Esempio CIRA: Aurelio Telegram Coach M1-M4: Foundation 26 file Python via full loop
Existing codebase (brownfield)
/gsd:map-codebase → /gsd:new-project --auto → phases → execute → verify → ship
Tempo stimato: 2-4h mapping, poi standard loop — Esempio CIRA: Mercurio MVP: /gsd:quick execute-phase deadline 7/5
Quick ad-hoc task
/gsd:quick [--discuss] [--research] [--validate] [--full]
Tempo stimato: 15min-2h dipendente da flags — Esempio CIRA: Minor fixes, utility functions, docs updates
Debug session
/gsd:debug → gsd-debugger investigates → gsd-code-fixer patches → verify
Tempo stimato: 30min-2h — Esempio CIRA: Issue isolation e root cause analysis
