Morph Compact 0.2.8

Context compaction veloce: 25-33k tok/s, +0.6% SWE-Bench Pro, sub-2s. Sostituisce il summarizer interno di Claude Code.

Quando usarlo: sessioni lunghe Claude Code che raggiungono il limite di contesto. Vuoi compaction rapida e di qualità superiore senza perdere catena di lavoro.
NON usarlo se: Non hai una Morph API key (a pagamento), o lavori sempre in sessioni brevi (<50% contesto).

Quick Start

# Install dal marketplace Claude Code
/plugin marketplace add morph/morph-claude-code-plugin
/plugin install morph-compact@morph

# Configura API key (interattivo)
/morph-compact:install

# Quando contesto pieno
/compact morph

API key salvata in ~/.claude/morph/.env (o env MORPH_API_KEY). Plugin aggiunge istruzioni a ~/.claude/CLAUDE.md per saltare il summarizer interno quando Morph gestisce.

Come funziona

Hook 3 punti del ciclo Claude Code:

  1. PreCompact — Morph Flash Compact a 33k tok/s comprime il contesto
  2. PostCompact — risultato Morph iniettato nel context status
  3. SessionStart — su resume/compact, Claude legge il contesto compresso da Morph

Prompt-injection nel CLAUDE.md istruisce il summarizer Claude a produrre solo un sentinel text, lasciando spazio a Morph. Non sempre rispettato — se fallisce, plugin lo detecta e ti propone /compact morph di nuovo (cache hit, no extra API call).

Comandi

/morph-compact:install
Cosa faSetup interattivo: prompt API key, salva in ~/.claude/morph/.env, aggiunge compact instructions al CLAUDE.md globale.
QuandoPrima volta dopo install plugin.
/morph-compact:uninstall
Cosa faRimuove instructions dal CLAUDE.md e cancella .env. Plugin resta installato.
/compact morph
Cosa faForza compaction via Morph. Claude vede l'istruzione "morph" e output solo sentinel, lasciando Morph summary prevalere.
QuandoManuale, prima del limite di contesto. O dopo failure auto-compact.
GotchaSu sessioni molto grandi prima compaction può fallire prompt-injection; ritry su contesto già ridotto è quasi sempre OK.

CIRA Context

CIRA Setup

API key necessaria

Morph è SaaS a pagamento — pricing su morphllm.com. Vale per build session ≥3h. Pagamento via Stripe.

CIRA Trade-off

Quando NON usarlo

Sessioni brevi (Telegram coach, scripting da 30min) non beneficiano. Lì il summarizer Claude basta. Riserva Morph per sessioni Mercurio/dashboard build da molte ore.

CIRA Stack

Pairing con claude-mem

claude-mem salva memorie persistenti tra sessioni; Morph comprime DENTRO la sessione corrente. Sono complementari, non sostituiti.

Gotchas & Anti-pattern

  • NO modo di disabilitare il summarizer Claude default (limitazione hook) — il plugin lo aggira via prompt-injection nel CLAUDE.md, non sempre seguito.
  • Su sessioni "molto grandi" prima compaction può ignorare istruzioni e produrre summary Claude completo. Soluzione: secondo /compact morph su contesto ridotto.
  • API key in chiaro nel .env — non committarlo in repo.
  • Plugin NON sostituisce Morph MCP (warpgrep + fast apply). Per stack completo installa anche MCP.
  • Conflitti con altri plugin compaction (es. context-mode). Tieni uno solo attivo.
