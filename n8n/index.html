n8n fair-code

Workflow automation per technical team. Code-when-needed flexibility + no-code speed. 400+ integration, AI-native via LangChain, self-host o cloud.

Quando usarlo: workflow ricorrente cross-service (Gmail → Slack → DB → AI agent → email reply). Self-host per privacy / air-gapped. Alternativa Zapier/Make ma open-source.
NON usarlo se: Workflow è one-shot script Bash/Python. Per orchestrazione AI agent semplice usa Claude Code skill direttamente.

Quick Start

# Instant try via npx
npx n8n

# Docker (recommended self-host)
docker volume create n8n_data
docker run -it --rm --name n8n -p 5678:5678 \
  -v n8n_data:/home/node/.n8n \
  docker.n8n.io/n8nio/n8n

# Editor
open http://localhost:5678

Cloud: app.n8n.cloud. Docs: docs.n8n.io.

Feature

  • Code-when-needed: JavaScript/Python custom, npm packages, o visual interface
  • AI-Native: agent workflow LangChain con tuoi data + modelli
  • Full control: self-host fair-code OR cloud
  • Enterprise: permission avanzati, SSO, air-gapped
  • Community: 400+ integration, 900+ template pronti

CIRA Context

CIRA Decision

Quando preferirlo a skill CC

Skill Claude Code (Aurelio, Mercurio) sono ottime per orchestrazione AI-pesante con prompt complessi. n8n è meglio per workflow lineari cross-service deterministic (Gmail trigger → check rules → CRM update → Slack notify) senza AI in mezzo.

CIRA Use case

Backend automation MVP cliente

Per MVP cliente Valerio (mail), Mercurio (mail demo) — workflow backend post-MVP può migrare a n8n self-host così cliente NON dipende da Claude API in produzione.

CIRA Stack

AI-Native LangChain

n8n ha node LangChain nativi — puoi mettere Claude/GPT in mezzo a workflow senza scrivere Python. Trade-off: workflow visual è più rigido di skill Claude Code.

Gotchas & Anti-pattern

  • License "fair-code" Sustainable Use — NON è MIT/Apache. Leggi termini per commercial use.
  • Self-host richiede infra (Docker + volume + backup) — non è zero-config come Zapier.
  • Workflow JSON portable, ma credenziali NO — re-auth dopo migrazione.
  • Cloud free tier ha limit execution/mese. Stima carico prima.
  • AI node hanno cost API esterni (OpenAI/Anthropic/etc.) — monitora con Codeburn + provider dashboard.
  • Per orchestrazione AI agent complessa, valuta prima Claude Code skill / Agent Teams — n8n può essere overkill.
