Caveman skill/plugin

Output Claude in stile uomo delle caverne. Taglia ~75% token, mantiene 100% accuratezza tecnica.

Quando usarlo: sessioni lunghe dove output prolisso brucia contesto e costi. Coaching iterativo, debug a ping-pong, build session multi-ora. Brain ancora grosso, bocca piccola.
NON usarlo se: output deve essere mostrato a clienti, sei in modalità docs/PR (Claude tiene normal mode su code/commit/security comunque), o stai facendo content writing dove il tono conta.

Quick Start

Una volta installato (skill o plugin), attiva al volo:

/caveman           # full mode default
/caveman lite      # solo drop filler/articoli
/caveman full      # frammenti, sintesi pesante
/caveman ultra     # telegrafico
/caveman wenyan    # cinese classico — ancora più corto
stop caveman       # torna normale
normal mode        # idem

Persiste per turno e sessione finché non revochi. Hook SessionStart può armarlo automatic — controlla ~/.claude/hooks/.

Livelli

lite
Cosa faToglie filler ("just", "really", "basically") e pleasantries ("happy to help"). Articoli restano.
QuandoVuoi tono diretto ma testo leggibile a voce alta.
full (default)
Cosa faFrammenti OK. Articoli via. Sinonimi corti (fix non "implement solution for"). Pattern: [thing] [action] [reason]. [next step].
QuandoModalità standard sessione di lavoro. Migliore tradeoff verbosità/leggibilità.
ultra
Cosa faTelegrafico. Tre parole frasi. Tabelle preferite a prosa. Solo info hard.
QuandoStatus check ricorrenti, monitor heartbeat, healthcheck output.
wenyan
Cosa faCinese classico: ancora più denso semantico. Sperimentale.
QuandoTest pure su context-saving. Non per uso quotidiano.

Comandi sessione

/caveman
Cosa faAttiva caveman mode al livello default (full).
EsempioUser digita /caveman → tutte risposte successive in stile caveman.
/caveman <level>
Cosa faSwitch livello senza disabilitare: lite | full | ultra | wenyan.
stop caveman
Cosa faDisattiva. Modalità normale ripristinata immediatamente.
See also"normal mode" (sinonimo)

Auto-clarity (drop automatico)

Caveman non si applica a contesti dove la chiarezza è critica:

  • Security warning — output verbose per evitare malintesi
  • Irreversible action confirmation — git push --force, rm -rf, drop table
  • Multi-step sequence dove l'ordine frammentato rischia di confondere
  • User chiede di chiarire / ripete la stessa domanda — segnale che caveman ha tagliato troppo
  • Code / commit message / PR body — sempre scritti in modalità normale

Dopo la parte critica, caveman riprende.

CIRA Context

CIRA Uso

Sessioni Telegram bot + coach

Caveman è il tono standard nelle sessioni Aurelio + Telegram dove user invia messaggi rapidi e si aspetta risposte rapide. Ottimizza il costo Opus 4.7 (1M ctx) abbattendo gli output di 3-4×.

CIRA Integration

Hook SessionStart

Il hook ~/.claude/hooks/ emette il system-reminder CAVEMAN MODE ACTIVE — level: full a ogni session start. Persiste finché non dici "stop caveman".

CIRA Benchmark

Risparmio osservato

Stima: 60-75% token output in sessioni coaching/build. Cache prompt warm rimane, output cut riduce solo egress. Cost saving stimato a fine giornata: ~$3-8 su Opus heavy.

Gotchas & Anti-pattern

  • Caveman non droppa info tecnica — droppa SOLO filler. Se output corretto manca dettagli, probabilmente non è caveman: è il modello che non sa.
  • Codice in caveman è sempre formattato normalmente. Mai abbreviare nomi variabili o saltare parentesi per "essere caveman".
  • Quando user chiede "puoi spiegare meglio?" → drop caveman per quella risposta, poi riprendi.
  • Commit message: caveman non si applica. Sempre Conventional Commits completi.
  • Plugin/skill: installa una versione sola (eviti doppi system-reminder che possono confliggere).
