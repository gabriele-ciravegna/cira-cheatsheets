PAI v5.0.0 · algo v6.3.0

Personal AI Infrastructure di Daniel Miessler. "Life Operating System": daemon Pulse (dashboard :31337), identità DA (Digital Assistant), Algorithm Current State → Ideal State, ISA primitive.

Quando usarlo: vuoi un sistema personale unificato che gira sopra Claude Code per gestire vita + lavoro. 45 skill, 171 workflow, 37 hook, privacy strutturale via containment zones.
NON usarlo se: Vuoi solo dev tool (PAI è life-OS). O hai già un sistema custom incompatibile (PAI è invasivo del filesystem).

Concetti core

DA (Digital Assistant)
Cosa faIdentità persistente che ricorda chi sei, cosa vuoi, come lavori. Ogni skill/workflow lo consulta.
ISA (Ideal State Articulation)
Cosa faPrimitive universale: descrive lo stato ideale di X (un progetto, una vita, un'azienda). Algorithm naviga dal Current State al Ideal State.
Pulse daemon
Cosa faUnified daemon che gira in background. Dashboard localhost:31337. Aggrega skill running + workflow + telemetria vita (sleep, calendar, focus).
Containment Zones
Cosa faPrivacy strutturale: contesto sensibile isolato in zone. Skill normale non vede zone "high-trust" senza esplicito grant.

Componenti v5.0

  • 45 skill — ricche, organizzate per dominio (productivity, security, content, comms, etc.)
  • 171 workflow — orchestrazioni multi-step pre-configurate
  • 37 hook — punti di estensione su lifecycle session/file/event
  • Packs — bundle skill+workflow per use case specifici
  • Pulse — daemon + UI :31337

Algorithm v6.3.0

Loop a 7 fasi che porta da Current State a Ideal State:

  1. Classify — quale modo (advisor / coach / executor) e quale tier (quick / standard / deep)?
  2. Frame — definisci ISA per il problema
  3. Map — Current State vs Ideal State
  4. Bridge — gap, ostacoli, risorse
  5. Sequence — ordine azioni
  6. Execute — fai
  7. Verify — sei più vicino all'Ideal State?

Classifier-driven: PAI decide quale modo+tier in base al prompt user.

Gotchas & Anti-pattern

  • PAI v5 è different system da v4 — NON è patch, è nuova architettura. Migration guide obbligatorio.
  • Pulse daemon su porta :31337 — conflitto possibile con altri servizi. Verifica lsof -iTCP:31337 prima di install.
  • Containment zones non-banali da configurare — leggi privacy guide.
  • CIRA: NON installare PAI sulla macchina principale prima di chiudere il piano workspace. Test su Mac portatile prima.
  • One-liner install esegue shell remoto — review install.sh prima di lanciare.
