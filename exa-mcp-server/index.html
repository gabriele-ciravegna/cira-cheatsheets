Exa MCP Server hosted

Connetti AI assistant alle capacità search di Exa: web search, code search, company research. Server MCP hosted: https://mcp.exa.ai/mcp.

Quando usarlo: ricerca web semantica (Exa è search neural — non keyword Google), code search GitHub semantico, company research per outreach/competitor analysis.
NON usarlo se: WebSearch built-in basta (free tier Anthropic). Exa è a pagamento a richiesta. Per GitHub specifico vedi gh search CLI.

# Claude Code
claude mcp add --transport http exa https://mcp.exa.ai/mcp

# API key da exa.ai/dashboard
# Configura nel client (vedi sezione install)

Hosted — no install locale. Solo API key da dashboard.exa.ai.

Tool MCP

web_search_exa
Cosa faSearch neurale sul web. Ritorna URL + snippet + content estratto.
QuandoRicerca semantica (similar pages, neural query). Più potente di Google su intent.
Esempio"Italian Renaissance palazzo SaaS competitors with virtual tour feature"
web_fetch_exa
Cosa faFetcha URL specifica e ritorna markdown pulito (no menu/ads/cookie).
QuandoAlternativa a Firecrawl scrape. Single-page extract.

Tool aggiuntivi (varia per piano): code search GitHub semantico, company research (LinkedIn/website parse), find-similar pages.

Install per client

  • Claude Code: claude mcp add --transport http exa https://mcp.exa.ai/mcp
  • Cursor: ~/.cursor/mcp.json → {"mcpServers": {"exa": {"url": "https://mcp.exa.ai/mcp"}}}
  • VS Code: .vscode/mcp.json → {"servers": {"exa": {"type": "http", "url": "https://mcp.exa.ai/mcp"}}}
  • Claude Desktop: native connector (Settings → Connectors → Exa → +)

CIRA Stack

Pairing strumenti

Firecrawl (testo strutturato) + designlang (design system extraction) + Exa (semantic discovery) = stack completo per audit competitor. Output va in ~/WORK/SITI/ispiration/<sitename>/ (vedi memoria feedback_site_clone_pipeline.md).

Gotchas & Anti-pattern

  • API key richiesta — non hosted gratis come WebFetch built-in Claude.
  • Search neural ≠ keyword: query troppo specifica peggiora risultati. Usa frase descrittiva, no "AND OR".
  • NON è sostituto di Google Search per news real-time — Exa è ottimizzato per evergreen + semantic similarity.
  • Hosted endpoint → richiede internet stabile. No offline.
  • CIRA: leggi memoria ordering (gh prima, Exa dopo) per evitare costi.
