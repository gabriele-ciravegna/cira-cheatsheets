Codex Plugin for Claude Code openai

Usa Codex (modelli OpenAI o1/o3) DENTRO Claude Code. Code review indipendente, delega task in background, adversarial challenge.

Quando usarlo: serve seconda opinione su decisione critica (review indipendente, no contamination con tuo plan). Delegare task lungo a background mentre lavori. Adversarial test su architettura.
NON usarlo se: Non hai ChatGPT subscription o OpenAI API key. Se vuoi review interno Claude → compound-engineering è più ricco.

Quick Start

# Marketplace + install
/plugin marketplace add openai/codex-plugin-cc
/plugin install codex@openai-codex
/reload-plugins

# Setup (check Codex installato + login)
/codex:setup

# Se Codex missing:
npm install -g @openai/codex
!codex login

# First run
/codex:review --background
/codex:status
/codex:result

Requisiti: Node 18.18+, ChatGPT subscription (anche free) o OpenAI API key. Usage conta su Codex limits.

Comandi

/codex:review
Cosa faCode review normale read-only via Codex. Stessa qualità di /review dentro Codex.
QuandoPR review indipendente. Vuoi opinion fuori dal tuo Claude context.
/codex:adversarial-review
Cosa faSteerable challenge review: Codex prova a rompere/disconfermare la tua architettura/diff.
QuandoDecisioni critiche reversibili-difficili. Pre-merge feature ad alto rischio.
/codex:rescue
Cosa faDelega task substanziale a Codex via shared runtime. Background job.
QuandoClaude bloccato/saturato, vuoi seconda implementazione, root-cause analysis profonda.
/codex:status
Cosa faMostra background job Codex attivi + stato.
/codex:result
Cosa faRitorna output del background job più recente (o specifico per id).
/codex:cancel
Cosa faCancella job background in corso.
codex:codex-rescue (subagent)
Cosa faSubagent disponibile in /agents per delega complessa, non solo review.
QuandoMulti-step research/debug che vuoi isolato dal main context.

Use Cases

  • Second opinion architecturale — disegno feature in Claude, /codex:adversarial-review per stress-test prima di committare
  • Long-running research — /codex:rescue con prompt esplorativo, continuo a lavorare in foreground, /codex:result dopo
  • PR review indipendente — Claude scrive PR, Codex review (no bias)
  • Stuck debug — Claude in loop su un bug, Codex con prospettiva fresca

Quando preferirlo a Agent Teams

Agent Teams (Sonnet+Sonnet) per debate interno modello. Codex Plugin per second opinion cross-modello. Diverso costo, diverso valore.

Gotchas & Anti-pattern

  • Codex CLI deve essere già installato + loggato. /codex:setup guida ma può fallire su network restrictions.
  • Background job → ricordati di chiamare /codex:result prima di /clear sessione, altrimenti perdi output.
  • Adversarial review può essere troppo aggressivo su prototipi — usa --background e leggi con calma.
  • NO usarlo per task triviali — overhead OpenAI auth + runtime + cost. Riserva a casi dove vale.
  • Tokens consumati su Codex limits, non Anthropic.
