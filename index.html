CIRA Cheatsheet Library

Nove cheatsheet HTML, duecento­ottantadue comandi, una bussola per decidere cosa usare e quando.

A cosa serve questa biblioteca: quando costruisci qualcosa di nuovo — un sito, un agente Telegram, un MVP — hai a disposizione tanti strumenti diversi (GSD, gstack, superpowers, ruflo, ecc.). Ognuno fa qualcosa di diverso. Qui c'è la cartina per non perdersi.

Che cos'è, in due righe

Ogni cheatsheet copre un tool dello stack di lavoro. Apri quello che ti serve, leggi solo la sezione che ti interessa. Ogni comando ha 5 cose: cosa fa, quando usarlo, esempio reale, vedi anche, trappola da evitare.

I 9 strumenti non si escludono a vicenda — si combinano. La pagina Workflows CIRA mostra le sequenze già usate (Mercurio, Aurelio, dashboard, MVP clienti).

Se non sai da dove partire, leggi la sezione Quando uso cosa qui sotto.

Le 9 librerie

Tocca una card per aprire la cheatsheet. Tutte hanno la stessa struttura: cosa fa, quando, esempio, gotcha.

GSD 1.42.2

Get Shit Done. Sistema spec-driven: spezzi il progetto in milestone → fasi → task atomici. Ogni fase ha audit trail, plan, verify, ship. 88 comandi · 33 agent specialist · 12 categorie.

Quando: progetto vero (≥3 file), serve audit trail, voglio sapere cosa è stato fatto e perché

Gstack

Ship-first toolkit. Stessa idea di GSD ma più rapida: plan → pair-agent → ship → retro → tune. Meno audit, più "facciamo e vediamo". 51 skill installate.

Quando: MVP da consegnare in 1-2 sessioni, velocità > documentazione

Superpowers

TDD + brainstorming. Approccio di Obra: prima brainstormi, poi scrivi i test (red), poi il codice che li fa passare (green), poi pulisci (refactor). 11 skill.

Quando: codice critico che deve essere testato bene

Ruflo

Sandbox A/B. Crea N varianti parallele dello stesso pezzo, le valuta, sceglie la migliore. Utile per scelte di UX/design ambigue. 20+ comandi.

Quando: non sai quale approccio funziona meglio, vuoi vedere alternative

Compound Engineering 3.8.2

Plugin Claude Code ce-*. Filosofia: ogni cycle migliora i prossimi (review continuo, learnings salvati). Ha 48 reviewer specialisti (correctness, security, performance, ecc.). 20 cmd + 48 agent.

Quando: lavoro in team, PR review continuo, multi-perspective

Agent Teams + Subagents

Multi-agent built-in di Claude Code. Due modalità: Teams (gli agent si parlano, debate) e Subagents (agent isolati, ritornano risultato). 3 pattern principali.

Quando: decisioni multi-prospettiva, task indipendenti paralleli

CIRA Custom Stack

Skill custom tue. orchestra (router), /costruiamo (build trigger), skill-creator-cira, mentori (Randolph, Bezos, Jobs…), pensiero strategico (Aristotle, Sinek). 14 skill core.

Quando: ogni giorno. È il tuo sistema nervoso

Claude Code 2.x

CLI ufficiale Anthropic. Come si avvia, slash command, hooks, skill, MCP, sub-agent, settings, modalità avanzate. 71 comandi · estratto da docs ufficiali + 6 tutorial YT (Saraev 6h, Ali Abdaal, productive-dude).

Quando: setup nuova macchina, "come si fa X?", scoprire feature nascoste

Bash + macOS

Sostrato di tutto. 66 comandi shell quotidiani: cd, find, grep, curl, lsof, launchctl, brew, jq, open, pbcopy, ecc. Più 15 shortcut zsh + automazione cron/launchd.

Quando: terminale aperto = sempre. Healthcheck post-blackout, debug servizi

Glossario per chi non è tecnico

I termini ricorrenti

CLI = "Command Line Interface", il terminale dove digiti comandi. claude è una CLI. git è una CLI.

Skill = un pezzo di istruzioni riusabile che Claude carica quando serve. Es: brainstorming è una skill.

Slash command = comando che inizia con / dentro Claude Code. Es: /gsd-new-project.

Hook = script che parte automaticamente in certi momenti (es: a inizio sessione, prima di salvare un file).

Agent / sub-agent = un "Claude più piccolo" che fai lavorare in parallelo su un task specifico, isolato dalla tua sessione principale.

Repo / repository = una cartella di codice tracciata con Git. Può essere su GitHub.

Deploy = mettere online. Es: deploy su Vercel = pubblica il sito su cira-dashboard.vercel.app.

MCP = "Model Context Protocol", il modo standard con cui Claude si collega ad altri servizi (Google Drive, Telegram, ecc.).

TDD = "Test-Driven Development". Prima scrivi il test (che fallisce), poi il codice che lo fa passare.

launchd = il sistema di macOS che fa partire programmi all'avvio. Il bot Telegram gira così.

cron = il modo classico unix per programmare task ricorrenti (es: ogni 5 min).

Per chi non sa

Che cosa è "Claude Code"?

È un'app da terminale che ti permette di parlare con Claude come se fosse un collega che apre i tuoi file, modifica codice, gira comandi. Non è il sito chat. È un'altra cosa: lavora nei tuoi progetti reali.

Perché tutti questi tool?

Ognuno è bravo in una cosa diversa. GSD è metodico (audit trail). gstack è veloce (ship-first). superpowers spinge i test. ruflo prova alternative. compound-engineering fa review di squadra. cira-custom sei tu.

Da dove parto?

Apri Quando uso cosa qui sopra. Trova la riga che descrive cosa stai facendo. Segui l'azione a destra.

Oppure: lancia /costruiamo X e Claude ti fa 10-15 domande per scegliere il path giusto.

Cosa è /costruiamo?

È una skill CIRA. Quando dici "facciamo X", Claude ti fa un'intervista lunga (chi è il cliente? cosa vuole? quanto tempo? quale brand?) e poi propone 1-3 path possibili tra GSD, gstack, superpowers, ruflo. Tu scegli. Lui esegue.
