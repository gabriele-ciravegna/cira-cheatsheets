Quick Start — Distinzione chiave

Teams: agent si parlano. Pattern debate/discuss.
Subagents: agent isolati. Pattern Task()-tool, ritornano risultato.

Vuoi…UsaPerché
Task paralleli indipendenti, result-only Task() (Subagent) Più veloce, meno costoso, isolamento context
Debate, consensus, multi-prospettiva TeamCreate() (Teams) I teammate si parlano direttamente, convergono
Task semplice <10 min single-view Main thread diretto Overhead Teams non vale la pena

Subagent isolato (Task tool)

Task({
  subagent_type: 'general-purpose',
  model: 'haiku',
  prompt: '...self-contained instruction, zero external context...',
  description: 'Email parser security audit',
  run_in_background: true
})

Team creation

TeamCreate({name: 'board', members: ['ceo', 'cto', 'cfo']})

Oppure via natural language: "crea team con N teammates per X"

Regola d'oro: Subagents per width (molti task indipendenti in parallelo) — Teams per depth (pochi agent, discussione complessa e convergenza).

Subagents (Task tool, result-only)

Task({subagent_type, prompt, description, …})
Cosa fa Spawn agente isolato, esegui prompt self-contained, ritorna risultato finale
Quando Ricerca indipendente, audit parallelo, code-review isolato, source-specific investigation
Return agent result summary (lossy per task >10KB)
See also TeamCreate per debate
Subagent NON ha memoria. Output lossy — usa per result-only.

Parametri Task()

ParametroTipoDescrizione
subagent_typestr general-purpose | researcher | code-reviewer | security-expert
promptstr Instruction self-contained — ZERO context da main thread
descriptionstr Human-readable task title (Agent View lo mostra)
modelstr haiku | sonnet | opus (default: auto-route)
run_in_backgroundbool true per non bloccare main (default: false)

Esempio: audit parallelo su 4 componenti

// Tutti in background, main thread non blocca
Task({
  subagent_type: 'researcher',
  model: 'haiku',
  description: 'Email parser security audit',
  prompt: 'Audit security vulnerabilities in email parsing. Focus: injection, header spoofing, MIME abuse. Return JSON: [{finding, severity, recommendation}]',
  run_in_background: true
})

Task({
  subagent_type: 'code-reviewer',
  model: 'haiku',
  description: 'Auth module review',
  prompt: 'Review auth module for session fixation, CSRF, token expiry. Return prioritized issues list.',
  run_in_background: true
})
Max 8–10 Task in 1 batch — oltre rischi rate limit e context burn.

Agent Teams (debate, collaborative)

TeamCreate({name, members, model, …})
Cosa fa Crea persistent team di N agent che collaborano via shared task list + direct messaging
Quando Board debate, PR multi-reviewer, complex refactor, hypothesis testing, go/no-go decisions
Lifecycle Lead → crea team → approval → teammates discuss → converge → result
See also Task() per non-debate work
Agent SI PARLANO. Cost linear con N members. Max ~8 members.

Parametri TeamCreate()

ParametroTipoDescrizione
namestr Team identifier (es. board, pr-reviewers, hypothesis-test)
memberslist[str] Teammate persona (es. cto, cfo, cmo, product-manager)
modelstr Model per teammates (default: auto-route)
lead_modelstr Model per lead (default: sonnet)

Esempio: PR review board

TeamCreate({
  name: 'pr-review-board',
  members: [
    'backend-specialist',
    'frontend-expert',
    'security-reviewer',
    'performance-auditor'
  ],
  model: 'sonnet'
})
Lead prompts team (natural language)
Cosa fa Main agent (lead) invia messaggio al team. Team discute, coordina, ritorna consensus.
Pattern Lead: "Crea team per investigare bug X con 3 hypothesis" → Team: ogni teammate testa (parallel) → Team: discutono, convergono → Lead: riceve consensus

Lifecycle & Control

Subagent lifecycle

  • 1Main agent spawns Task({prompt, …})
  • 2Subagent isolated, execute prompt
  • 3Subagent returns result
  • 4Main agent receives summary, proceed

Team lifecycle

  • 1Lead prompts "create team with N members for X"
  • 2TeamCreate spawns persistent team + lead
  • 3Lead proposes plan
  • 4Team discute, converge on approach
  • 5Lead approves or redirect
  • 6Teammates execute, self-coord
  • 7Team returns consensus result
  • 8Lead can continue or close team

Limitazioni experimental (v2.1.131)

  • No pause/resume mid-team
  • Max ~8 teammates per team (rate limit)
  • Team lifecycle is linear (no branching logic)
  • Inter-agent messaging visible to lead only
  • Voting/consensus NOT automatic — lead must interpret results

Multi-agent Patterns

Parallel research (Subagents)
9× Task() in parallel, ognuno ricerca source indipendente.
Quando: Multi-source competitive analysis, audit 10 files, generate 4 variants.
Model strategy: Haiku (cost) ×9 → Sonnet ×1 (synthesis)
C-suite board (Teams)
5–7 agent debate, converge on decision.
Quando: Architectural decision, pricing strategy, go/no-go call.
Model strategy: Sonnet lead, Haiku/Sonnet teammates
Specialist chain (Subagents sequenziali)
Task A → output → Task B (include A output) → output → Task C synthesis.
Quando: Multi-stage, specialist each stage.
Model strategy: Haiku chain, Sonnet final synthesis
Hypothesis testing (Teams debate)
Team testa 3+ hypothesis in parallelo, debate convergence.
Quando: Bug root-cause, feature feasibility study.
Model strategy: Sonnet per debate quality
CIRA Tavolo Consiglieri (Teams scale)
67 mentori in 3 mode: consiglio (serial, 1 answer) | parallel (N task contemporanei) | concistoro (vote/consensus).
Quando: CIRA coaching decisions, multi-perspective life planning.
Model strategy: Opus lead (reasoning depth), Sonnet teammates

Cost model

Subagents 9× parallel

9× Haiku (research) → 1× Sonnet (synthesis)

Team 5 members

5× Sonnet (debate, 3–5 rounds) — costo significativamente più alto

CIRA model routing: Opus solo registry/canonical — Sonnet per debate Teams — Haiku per Subagents execution. Mai Agent parallel (68% burn rilevato).

Workflows End-to-End

Multi-source research (Subagents)

Pattern: Parallel research → Sonnet synthesis
Trigger tipico: "Analizza 5 competitor su pricing, feature, positioning" 
// Step 1: spawn N subagents in background
const sources = ['competitor-A', 'competitor-B', 'competitor-C', 'competitor-D', 'competitor-E']

sources.forEach(s =>
  Task({
    subagent_type: 'researcher',
    model: 'haiku',
    description: `Research ${s}`,
    prompt: `Analizza ${s}: pricing page, feature list, positioning statement. Return structured JSON {pricing, features[], positioning}.`,
    run_in_background: true
  })
)

// Step 2: Sonnet synthesis dopo raccolta risultati
Task({
  subagent_type: 'general-purpose',
  model: 'sonnet',
  prompt: 'Sintetizza i 5 competitor analysis in white space opportunities per CIRA.'
})

Anti-patterns ⚠

  • Mai mettere team_name su Task() — error (workaround v2.1.128, deprecated in v2.1.131)
  • Mai >8 agent paralleli subagent in 1 batch — rate limit + context burn
  • Subagents ritornano summary lossy — MAI per task con stato condiviso tra agent
  • Teams debate senza approval gate = risk — sempre far approvare al lead il piano del teammate
  • Mescolare Team membership durante sessione = instabile — crea team statici, non mutarli mid-session
  • Subagent prompt context-dependent = fallimento — assicura prompt completamente self-contained, zero riferimenti a context del main thread

Gotchas

  • 🔇 Subagent NON ha memoria. Ogni Task parte da zero. Non passare contesto implicito — tutto deve stare nel prompt.
  • 📉 Output lossy su task grandi. Se il task produce >10KB di output, il summary è compresso. Mai usare Subagents per stato condiviso.
  • 💸 Teams è costoso. Ogni teammate = full session. 5 teammate × 3 round debate = costo significativo. Usa Haiku per teammate quando possibile.
  • 🕽 Voting non automatico. Il consensus nei Teams non è auto-calcolato — il lead deve interpretare i risultati e sintetizzare.
  • ⛔ No pause/resume. Una volta avviato un Team, non puoi metterlo in pausa e riprendere in sessione diversa (experimental v2.1.131).
  • 👁 Inter-agent chat visibile solo al lead. Il main thread vede solo il risultato finale del team, non i messaggi interni tra teammate.
  • 💥 68% context burn con Agent parallel. Test CIRA: mai usare Agent parallel su task lunghi — usa Subagents Task() invece.
Model routing CIRA: Opus solo registry/canonical — Sonnet per debate Teams — Haiku per Subagents execution. Mai Agent parallel (68% burn). Sweet spot main thread: 3–5 skill max.
