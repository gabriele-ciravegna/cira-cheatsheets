Gstack 2.x

Ship-first toolkit. Plan → pair-agent → ship → retro → tune.

Quando usarlo: Velocità prioritaria su audit trail. MVP ready in 1-2 sessioni. Path A2 in /costruiamo intervista.
NON usarlo se: Compliance audit obbligatorio (→ GSD), bug fix singolo (→ /gsd-debug)

Quick Start

Pipeline ship-first standard:

/gstack:plan → /gstack:pair-agent → /gstack:ship → /gstack:retro → /gstack:plan-tune

Workflow End-to-End

Ship MVP rapido
durata: 1-2 sessioni
Pipeline
  1. /gstack:plan — pianificazione
  2. /gstack:pair-agent — esecuzione pair-programming
  3. /gstack:ship — deployment
  4. /gstack:retro — retrospettiva
Design to Ship
durata: 2-3 sessioni
Pipeline
  1. /gstack:design-consultation — consulenza design
  2. /gstack:design-html — generazione HTML
  3. /gstack:design-review — review
  4. /gstack:pair-agent — coding
  5. /gstack:ship — deploy
